# v0.3 Journal, Tasks, and Schedule Plan

Date: 2026-07-03
Depends on: `v0.2 ActivityWatch Aggregation`

## Goal

Add local persistence and the first editable personal planning features: daily learning journals, simple tasks, and daily schedule blocks. This turns StudyPulse from a read-only activity dashboard into a lightweight daily workspace.

## Non-Goals

- Do not add AI-generated plans yet.
- Do not add recurring tasks or nested projects.
- Do not sync with external calendars.
- Do not build complex task filtering or search.

## Tasks

1. Add SQLite persistence.
   - Create a local SQLite database in `data/study-pulse.sqlite` by default.
   - Initialize tables on API startup.
   - Keep the data directory ignored by git.

2. Add journal API.
   - Get a journal by date.
   - Save or update a journal by date.

3. Add task API.
   - List tasks.
   - Create tasks.
   - Update completion, title, area, priority, and planning bucket.
   - Delete tasks.

4. Add schedule API.
   - List schedule blocks by date.
   - Create schedule blocks.
   - Update time range and title.
   - Delete schedule blocks.

5. Update frontend views.
   - Add editable Journal view.
   - Add task creation, completion, and deletion.
   - Add schedule block creation and deletion.

6. Verify.
   - Run backend import check.
   - Run frontend build.
   - Smoke test CRUD endpoints against the local API.

7. Publish.
   - Commit and push v0.3 changes.

## Acceptance Criteria

- The API creates the SQLite database automatically.
- Journal content persists after saving and reloading.
- Tasks can be created, completed, and deleted.
- Schedule blocks can be created and deleted for a date.
- The frontend build passes.
