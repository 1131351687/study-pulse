# v0.4 AI Summary and Planning Plan

Date: 2026-07-03
Depends on: `v0.3 Journal, Tasks, and Schedule`

## Goal

Add the first AI planning loop. The backend should build a compact daily context from ActivityWatch aggregation, journal content, tasks, and schedule blocks, then generate a structured daily summary and next-day plan suggestion. AI output is saved as a generated result, but suggested tasks and schedule blocks are not automatically applied.

## Non-Goals

- Do not let AI directly mutate tasks or schedules.
- Do not implement weekly summaries yet.
- Do not add long-term memory, embeddings, or search.
- Do not build a full secret manager yet.

## Tasks

1. Add AI configuration.
   - Support `mock`, `openai`, and `ollama` provider modes through environment variables.
   - Keep API keys out of public settings responses.

2. Add AI provider layer.
   - Add deterministic mock provider for local development and open-source demos.
   - Add OpenAI-compatible chat completions provider.
   - Add Ollama chat provider.

3. Add daily context builder and generation route.
   - Build context from journal, tasks, schedule, and activity summary.
   - Add `POST /api/ai/daily-plan`.
   - Store generated output in SQLite.

4. Update frontend.
   - Add Generate summary/plan button on Today page.
   - Render summary, topics, insights, suggested tasks, and suggested schedule.
   - Update Settings view to show current provider settings.

5. Verify.
   - Run backend import check.
   - Run frontend build.
   - Smoke test mock generation route.

6. Publish.
   - Commit and push v0.4 changes.

## Acceptance Criteria

- The app can generate a daily plan without external credentials using the mock provider.
- OpenAI-compatible and Ollama providers have a clear implementation path through env config.
- AI output is structured JSON and renders on the Today page.
- Suggested tasks and schedule blocks are displayed as suggestions only.
