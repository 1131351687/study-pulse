# v0.1 Local Runnable Skeleton Plan

Date: 2026-07-03
Spec: `docs/superpowers/specs/2026-07-03-study-pulse-design.md`

## Goal

Create the first runnable StudyPulse skeleton: a local FastAPI backend, a React/Vite frontend, a small shared project structure, development scripts, and documentation. This version proves that the project can run locally and gives later ActivityWatch, planner, and AI work a clean place to land.

## Non-Goals

- Do not implement real ActivityWatch aggregation yet.
- Do not implement AI provider calls yet.
- Do not implement persistent task, journal, or schedule CRUD yet.
- Do not add desktop packaging.

## Tasks

1. Create root project metadata and scripts.
   - Add root `package.json` with convenience scripts for frontend and backend.
   - Keep local data, secrets, dependency folders, and generated files ignored.

2. Scaffold backend in `apps/api`.
   - Add FastAPI app entrypoint.
   - Add config module with local defaults.
   - Add health route.
   - Add placeholder settings route.
   - Add requirements file.

3. Scaffold frontend in `apps/web`.
   - Add React + TypeScript + Vite setup.
   - Add basic app shell with navigation for Today, Journal, Tasks, Schedule, and Settings.
   - Add API client that calls backend health endpoint.
   - Add simple responsive styling.

4. Add documentation.
   - Add README with local startup instructions.
   - Document current v0.1 scope and next planned milestones.

5. Verify.
   - Install frontend dependencies.
   - Install backend dependencies if needed.
   - Run frontend build.
   - Run backend import or health smoke check.

6. Publish.
   - Commit v0.1 skeleton changes.
   - Push to `https://github.com/1131351687/study-pulse.git`.

## Acceptance Criteria

- `apps/api` contains a runnable FastAPI app.
- `apps/web` contains a buildable React/Vite app.
- The frontend can show backend health status when both services are running.
- README explains how to install dependencies and start both services.
- No secrets or local runtime data are committed.
