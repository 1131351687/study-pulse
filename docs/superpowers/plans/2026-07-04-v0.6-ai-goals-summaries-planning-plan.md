# v0.6 AI Goals, Summaries, and Planning Plan

Date: 2026-07-04
Depends on: `2026-07-04-study-pulse-ai-goals-schedule-design.md`

## Goal

Reshape StudyPulse around goal-driven AI planning and date-centered daily learning records. Replace the separate history page with a schedule page that acts as the main daily record view, narrow tasks to today's execution list, add learning goals, add AI summaries per date, and add goal-based AI planning for today and this week.

## Non-Goals

- Do not add automatic acceptance of AI-suggested tasks.
- Do not rewrite journal text bodies with AI content.
- Do not add long-term analytics, embeddings, or semantic search.
- Do not add cloud sync or external calendar integrations.

## Tasks

1. Extend the database and backend models.
   - Add `learning_goals`.
   - Add `ai_summaries`.
   - Add `ai_plans`.
   - Add `ai_plan_suggested_tasks` if needed for per-task acceptance tracking.

2. Add backend routes.
   - Add goals CRUD routes.
   - Add `POST /api/ai/summary` and `GET /api/ai/summary/{date}`.
   - Add `POST /api/ai/plan` and `GET /api/ai/plan/{date}/{goal_id}`.
   - Add one route to accept one suggested task into today's tasks.
   - Add a composed `GET /api/day-record/{date}` route for the schedule page.

3. Add backend AI logic.
   - Build a daily summary context from ActivityWatch, journal, schedule, and today's task execution.
   - Build a goal-driven planning context from the selected goal, recent day records, and recent AI summaries.
   - Keep mock provider support so the full flow works without external credentials.

4. Reshape frontend information architecture.
   - Remove the separate history page.
   - Turn schedule into the main date-centered learning record page.
   - Narrow tasks to today's task workflow.
   - Add learning goals page.
   - Add AI summary page.
   - Add AI planning page.

5. Wire AI summary interactions.
   - Generate summaries by date.
   - Show stored summaries for a selected date.
   - Display score, strengths, blockers, and improvements.

6. Wire AI planning interactions.
   - Select a goal.
   - Generate today's and this week's plan.
   - Show suggested tasks.
   - Allow manual acceptance of selected suggested tasks into today's tasks.

7. Verify.
   - Run backend import check.
   - Run frontend build.
   - Smoke test goals CRUD.
   - Smoke test AI summary generation.
   - Smoke test AI plan generation for a selected goal.
   - Smoke test day-record route.

8. Publish.
   - Commit and push the completed v0.6 changes.

## Acceptance Criteria

- The navigation no longer exposes a separate history page.
- The schedule page can browse dates and show a full daily learning record.
- The tasks page is focused on today's tasks.
- The user can create and manage one or more learning goals.
- The user can generate AI summaries per date.
- AI summaries are stored separately from journal text and shown together with the selected day.
- The user can generate AI plans per goal with daily and weekly scope.
- Suggested tasks remain optional until explicitly accepted.
- Accepted suggested tasks appear in today's tasks.
