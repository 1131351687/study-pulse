# v0.2 ActivityWatch Aggregation Plan

Date: 2026-07-03
Depends on: `v0.1 Local Runnable Skeleton`

## Goal

Add the first real data source: ActivityWatch. The backend should read today's local ActivityWatch events, aggregate them into useful app and title summaries, and expose a stable API that the Today page can render.

## Non-Goals

- Do not persist ActivityWatch snapshots to SQLite yet.
- Do not classify learning categories with AI yet.
- Do not read screenshots, OCR, or browser history directly.
- Do not require ActivityWatch to be running for the rest of the app to work.

## Tasks

1. Add backend ActivityWatch adapter.
   - Read buckets from ActivityWatch REST API.
   - Select likely window and browser buckets.
   - Read events for the current local day.
   - Aggregate durations by app and title.
   - Return graceful unavailable responses when ActivityWatch is down.

2. Add activity API route.
   - Add `GET /api/activity/today`.
   - Return date, availability, total seconds, top apps, top titles, and a message.

3. Update frontend API client.
   - Add typed `fetchTodayActivity` function.
   - Keep backend health check separate.

4. Update Today page.
   - Show tracked time from backend data.
   - Show top apps and top titles with simple duration bars.
   - Show a non-blocking ActivityWatch unavailable state.

5. Verify.
   - Run backend import check.
   - Run frontend build.
   - Start backend and request `/api/activity/today`.

6. Publish.
   - Commit and push v0.2 changes.

## Acceptance Criteria

- `GET /api/activity/today` never crashes when ActivityWatch is unavailable.
- When ActivityWatch is available, the API returns aggregated current-day durations.
- The Today page renders activity data or a clear unavailable message.
- The frontend build still passes.
